# Force alerts carry persistent team state; chat carries transient events

MTS already tells players things six ways (broadcast, direct print, pop text, rendered labels, GUI panels, Discord bridge). Factorio 2.1's `LuaForce::add_custom_alert` adds a seventh — persistent, map-anchored, dismissible — and adopting it without a rule for what goes where would make every announcement a judgment call. The rule: **alerts are reserved for a persistent state the team is currently subject to; anything that merely *happened* stays on chat/pop-text/sound.**

First (and for now only) user: team pause. A paused team's members previously got no notification at all — only the admin who ran the sweep saw a print — which is exactly the missable, currently-true condition alerts exist for. All pauses alert, with **source-distinct wording**: admin pauses carry the full admin-froze-you message, script pauses (the `pause_team` remote — MTS Dimension Warp pauses routinely as dock gameplay) carry neutral "base suspended" wording, so the alert never claims an admin action that didn't happen. The alert is removed on unpause (targeted `remove_alert{entity = anchor}` on a validity-checked anchor — never the empty filter, which clears a player's entire tray) and re-raised for members who join mid-pause. Pause state also leaves the game: additive mts-v1 events `on_team_paused`/`on_team_resumed` fire for every pause (unconditionally — consumers depend on them), and admin-source pauses are mirrored to the Discord bridge, so the channel that previously stayed blind during a pause no longer does. Rejected under this rule: buddy requests (already carried by per-member modal frames), milestones (transient celebrations, already on three channels), rival-overtake (transient, spam-prone).

Alerts need an entity anchor and MTS had no guaranteed per-team entity (the passive radar exists only when consumer mods request one), so each team gets a purpose-built hidden, inert, indestructible anchor — **ensured lazily at alert time on a currently-valid team surface, never trusted to exist**. Create-on-claim is impossible (no team surface exists yet at claim time) and no anchor survives contact with the consumer ecosystem: cave-collapse mods `die()` entities regardless of `protected`/`destructible`, Brave New MTS clears entity footprints near spawn, and Dimension Warp retires the original home surface on a team's first warp. The anchor is expected to die mid-life and is simply re-ensured on next use. The channel is gated by the `team_alerts_enabled` admin flag, default on, matching the mod's convention that comparable UX features are flag-gated; the flag never gates the mts-v1 events or bridge raise.
